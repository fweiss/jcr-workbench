JCRWorkbench Use Cases

The uses cases for JCRWorkbench are deliberated elaborated in the code than in this document. However, to remain sane, it is intended that this document cover at least the less obvious ones.

General

General use cases

Add node and property

This applies only to non-multi properties an the property type STRING.

  • Navigate to and select the parent node.
  • Right click it and choose "Add Node".
  • Enter node name and value (implict type is nt:unstructured).
  • Click OK.
  • Click "Add Property".
  • Enter node name and value (implict type is STRING).
  • Click OK.
  • Handle error reponses.

Effects

  • Given node and property displayed.
  • Given node and property tagged as new.
  • When saved, new node and property are persisted.

Create a new repository

This is a default behavior for a new workspace.

  • Start JCRW.
  • Enter a non-existent configuration (the default for clean build).
  • Click "Open".
  • The dialog box explains that a new repository is going to be created.
  • Click OK.
  • The default repository workspace is displayed.
  • The created repository can be opened
